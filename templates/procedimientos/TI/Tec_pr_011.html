1. OBJETIVO

Dar a conocer los mecanismos de gestión de firewall, su control y su monitoreo.

2. ALCANCE

Este procedimiento aplica para los funcionarios que por la naturaleza de su labor administran y gestión los recursos de la red.

3. RESPONSABLES

Recurso Tecnológico, Oficial de seguridad informática.

4. DEFINICIONES
  • RED: Es un conjunto de equipos conectados a un servidor por medio de cables, señales, ondas o cualquier otro método de transporte de datos, que comparten información (archivos), recursos (CD-ROM, impresoras, etc.), servicios (acceso a Internet, e-mail, Chat), etc. incrementando la eficiencia y productividad de las empresas.
  • FIREWALL: Es una parte de un sistema o una red que está diseñada para bloquear el acceso no autorizado, permitiendo al mismo tiempo comunicaciones autorizadas.
  • MECANISMOS DE SEGURIDAD: Un mecanismo de seguridad informática es una técnica o herramienta que se utiliza para fortalecer la confidencialidad, la integridad y/o la disponibilidad de un sistema informático.
  • I.P: Dirección de transito
5. DOCUMENTOS DE REFERENCIA
  • TEC-PR-013 Gestión De Redes
  • ISO 27001
6. DESCRIPCIÓN DEL PROCESO
6.1. Gestión de Dispositivos de Red
6.1.1. Acceso Al Firewalls
  • Responsable: Recurso Tecnológico
  • Registro: No aplica
  • Descripción: Se ingresa al firewall según la sede que se desea monitorear:
    • Sede elemento1: 192.168.0.173:4545
    • Sede elemento2 192.168.70.24:4545
    • Sede soledad 192.168.40.24:4545
    Los usuarios de la vpn se crean acorde con el procedimiento de gestión de usuarios tec-pr-004 se debe registrar y documentar la creación de estos usuarios.
6.1.2. Gestión De Firewalls
  • Responsable: Recurso Tecnológico
  • Registro: Informe de correo electrónico, TEC-D-022 Sitios web permitidos y bloqueados
  • Descripción: Para la administración del firewall se debe tener en cuenta cualquier cambio físico en la red, teniendo presente la red de origen y de destino para dicho cambio, si se debe crear una nueva red hay que contar con el administrador de t.i y el administrador de firewalls para su creación y su configuración de acceso.
    El firewall administra servicios de red configurados con los clientes: expresado en la actividad 1.4.
    Para el filtrado web y la aprobación de páginas web se debe realizar según el procedimiento tec-pr-002 soporte técnico
    Las paginas aprobadas se llevan en el registro: tec-d-022 sitios web permitidos y bloqueados
    Se monitorea el uso de la wan verificando que no exceda el ancho de banda indicado por el proveedor que oscila entre 9mb, en caso de saturación se revisa el tráfico de descargas y se realiza gestión sobre la ip del cliente que esté usando más tráfico en el momento.
    Se Realiza El Monitoreo 2 Veces En El Día En Los Horarios:
    • 8:00 a.m.
    • 5:00 pm.
    El encargado de monitoreo es el recurso tecnológico y las acciones realizadas quedan guardadas en el reporte que llega al correo electrónico, los correos verificados quedan en \\192.168.0.130\Confidencial_TI\Formatos Gestión T-I\Monitoreo Firewall.
6.1.3. Gestión De Switches
  • Responsable: Recurso Tecnológico
  • Registro: TEC-D-023 Segmentación Red Digicom
  • Descripción: Disponemos de Switches HP1920 capa 3 en cada sede debidamente configurado
    • Administrado por el Recurso Tecnológico se crean y administran las VLAN´s
    • Cada cliente cuenta o pertenece a una VLAN según el proceso.
    • En caso de necesidad por la producción se remite correo
    • Se realizan control de puertos, porque se tienen listas de MAC permitidas TEC-D-023 Segmentación Red Digicom.
    • Al conectar un equipo con una MAC diferente a las registradas bloquea el acceso.
6.1.4. Servicios De Red
  • Responsable: Recurso Tecnológico
  • Registro: No aplica
  • Descripción: El área de T.I. gestiona Los servicios de red que se requiere para cada uno de los diferentes clientes; estos servicios son:
    • Internet, Correo, Web mail, Ftp, ftps, VPN, Carpetas compartidas, Monitoreo de cámaras, Sitio Web de descarga de imágenes, Impresoras, Teléfonos I.P., Página web, Administración de Sistema de acceso electrónico, Firewall.
6.2. Actualización De Acuerdo De Servicio
  • Responsable: Recurso Tecnológico
  • Registro: No aplica
  • Descripción: Se realizan los ajustes necesarios de acuerdo a los requerimientos externos o internos.
