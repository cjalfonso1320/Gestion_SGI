1. OBJETIVO

Establecer las actividades relacionadas con el procedimiento de creación de copias de seguridad de la información procesada del cliente para dar cumplimiento del Sistema de Gestión Integrado

2. ALCANCE

Este procedimiento aplica a los datos capturados del cliente, correos del proceso, imágenes e información interna

3. RESPONSABLES

Personal del Proceso Recurso tecnológico, Oficial de Backup, Lideres de proceso y Oficial de seguridad informática

4. DEFINICIONES

Copias de Seguridad: Permite guardar con seguridad los datos procesados o entregados, con fin de protegerlos de riesgos potenciales

5. DOCUMENTOS DE REFERENCIA

Requisitos del Cliente

6. DESCRIPCIÓN DEL PROCESO
6.1. Realización de Copia de Seguridad
6.1.1. Ingreso Al Aplicativo
  • Responsable: Oficial De Backup
  • Registro: Correo De La Herramienta De Backup
  • Descripción: La consola Cell Manager de la aplicación Data Protector Manager se encuentra en una maquina asignada. Por este aplicativo se hace la programación de las copias, se estableció que se efectuaran copias de seguridad diarias (de lunes a viernes), semanales y mensuales las dos primeras se reciclan periódicamente y las mensuales se conservan como backup. Esto para las bases de datos y archivos del proceso.
    Para las máquinas virtuales se ejecuta mediante replicación a la sede Soledad que actualiza la copia cada 15 minutos.
    El último fin de semana de cada mes se efectuarán los Backups mensuales,de bases de datos y archivos de las aplicaciones. Incluye los logs de aplicaciones, carpetas y backups de dispositivos (Switches, routers y firewalls) y DVR'S (según procedimiento y acuerdo de servicio).
6.1.2. Programación y Realización De Backup
  • Responsable: Oficial De Backup
  • Registro: Reporte Herramienta De Backup
  • Descripción: El funcionamiento es automático de acuerdo a la programación. (Incremental: diario o semanal y Totales: mensuales) La aplicación envía un correo con el informe diario del resultado de las tareas a T.I. Este contiene la información acerca del inicio, sucesos, duración y errores que se presentan en la ejecución de tareas
6.2. Revisión De La Copia De Seguridad
  • Responsable: Oficial De Backup
  • Registro: No Aplica
  • Descripción: Se debe revisar el informe que se archiva en la ruta:\\192.168.40.146Confidencial_TI\Formatos Gestion T-I\Copias De Seguridad\Reportes Data Protector De acuerdo a este informe se revisarán las causas de las inconsistencias para generar las tareas nuevamente y lograr efectuarlas exitosamente.
6.3. Almacenamiento De Copias De Seguridad
  • Responsable: Oficial De Backup
  • Registro: No Aplica
  • Descripción: Los cartuchos activos se mantienen en los dispositivos de Backup que a su vez están en los centros de cómputo de las sedes Las cintas se almacenan en el sitio de custodia aislado con alarma y cámara.
6.4. Restauración De los Datos
6.4.1. Solicitud De Clientes
  • Responsable: Líder De Proceso Y Director De Procesos
  • Registro: No Aplica
  • Descripción: El líder del proceso o Director de procesos solicita por medio de un Ticket de Servicio los datos a restaurar al Oficial de Backup
6.4.2. Generación De Restauración De Copias De Seguridad
  • Responsable: Oficial De Backup
  • Registro: Tec-D-013 restauración De Copias De Seguridad
  • Descripción: Se registra en el formato el nombre del solicitante, área y fecha de solicitud, se procede a su restauración, Registrar en el formato los datos de restauración: datos a restaurar y la fecha de entrega
6.5. Tiempo De Conservación
  • Responsable: Oficial De Backup
  • Registro: No Aplica
  • Descripción: El tiempo de conservación se debe tener en cuenta según los acuerdos de servicios, los contratos y la experiencia en el manejo de dicha información.
6.6. Entrega De Backup Al Cliente
  • Responsable: Líder De Proceso
  • Registro: No Aplica
  • Descripción: En cumplimiento de los acuerdos de servicio con los clientes se realiza la entrega de la información.
6.7. Backup De Contingencia
  • Responsable: Líder De Proceso
  • Registro: No Aplica
  • Descripción: El líder del proceso debe realizar una copia semanal de todas las carpetas y tablas necesarias para el funcionamiento del aplicativo según política de carpetas y se guarda localmente en los equipos asignados al líder.
6.8. Backup De Correo
  • Responsable: Colaboradores Con Cuenta De Correo
  • Registro: No Aplica
  • Descripción: Los funcionarios deben abrir el software que está en el equipo asignado a cada uno, en C:\CorreoBackup y ejecutar CorreoBackup El aplicativo guarda la copia en la ruta \\192.168.0.141\BackupCorreo.
6.9. Pruebas De Restauración De Datos
  • Responsable: Oficial De Backup y Oficial de Seguridad Informatica
  • Registro: TEC-D-015 Pruebas de Restauracion Copias de Seguirdad
  • Descripción: Se realiza un programación tomando aleatoreamente un proceso, estas pruebas se realizan cada 3 meses. Se registra en el formato y se deja como evidencia de la restauración un acta donde se puestra el paso a paso.
