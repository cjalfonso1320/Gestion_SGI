3. RESPONSABLES

4. DEFINICIONES

 FTP: es un protocolo de transferencia de archivos entre sistemas conectados a una red TCP (protocolo de control de transmisión) basado en la arquitectura cliente-servidor, de manera que desde un equipo cliente nos podemos conectar a un servidor para descargar archivos desde el o para enviarle nuestros propios archivos independientemente del sistema operativo utilizado en cada equipo.  Archivo Plano: archivo de datos que se utiliza para el intercambio de información entre las diferentes aplicaciones.  Conciliación: registro consolidado de transacciones por convenios.  Log Bancario: es el registro de las transacciones de una fecha de recaudo.  Precinto: Zuncho de seguridad numerado que una vez colocado en el medio de transporte de documentos o valores no puede ser violentado ni roto excepto por el destinatario.  Tula: empaque en material de lona que permite el transporte seguro de documentos.  Servidorproceso: 192.168.70.137  Servidorcanal: 192.168.40.127

6.4 Recepción documental
  • Responsable: Auxiliar de proceso
  • Registro: No aplica
  • Descripción: Recibir los documentos de impuestos departamentales remitidos por el Banco, mediante tulas. Estas se organizan en orden de menor a mayor por número de Oficina. Se procede a cantar las tulas ante cámaras, mencionando de manera clara la oficina que remite la tula, el número de precinto, la fecha de recaudo, el convenio al cual pertenecen y la cantidad de documentos enviados; al desocupar la tula se muestra ante cámara y se canta como tula vacía lo cual certifica que no hay más documentos al interior de la tula Al terminar se realiza la devolución de las tulas vacías a la empresa de mensajería encargada de realizar la distribución de las mismas
  • Nota: Durante el proceso de apertura de tulas, es posible que lleguen documentos que no pertenecen al proceso o que pertenecen a otra ciudad de Digicom. Estos documentos se cantan ante cámara como documento mal remitido mencionando claramente la oficina que los envía y se entregan al líder de proceso para que realice el proceso correspondiente
